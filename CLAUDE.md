# 単語帳アプリ

## プロジェクトの目的
AI利用やプログラミングの初心者が、実際に手を動かしながら学ぶための単語帳アプリ。GitHub Pagesで公開しており、スマホからも利用する。

## 応答方針
- 日本語で簡潔に返答する。
- 初心者向けであることを踏まえ、専門用語を使う場合は簡単な補足を添える。

## 単語帳の運用
- `単語帳.html`(表示・クイズ)、`単語帳データ.js`(用語データ)、`単語帳文法データ.js`(言語別の文法クイズデータ)で構成される。
- 会話中にユーザーへ専門用語の意味を説明したときは、その用語がまだ`単語帳データ.js`のWORDS配列に登録されていなければ、末尾に`{ term: "用語", meaning: "意味" }`の形式で追加する(重複登録はしない)。
- 説明した用語がプログラミング言語名(例: Python, JavaScript)の場合は、WORDSへの追加時に`{ term: "言語名", meaning: "説明", type: "language", quizKey: "識別子" }`の形式にし、加えて`単語帳文法データ.js`のGRAMMAR_QUIZZESに同じ`quizKey`でその言語の文法クイズを追加する。データ形式は`{ name: "◯◯編", questions: [ { q, choices, answer(正解のindex), exp }, ... ] }`で、questionsは学習を進めるうえでの難易度順(やさしい→難しい)に並べる。週や単元でのサブカテゴリ分けはしない。これによりクイズ画面に「◯◯編」が自動的に出現する。
- 追加した場合は、その旨を一言ユーザーに伝える。
- ファイルを編集する前に必ず`git pull`を実行し、他の端末(スマホのクラウドセッションなど)で追加された最新の内容を取り込んでから編集する。これを怠ると、他端末での追加分を知らないまま古い内容で上書きしようとして`git push`が拒否される。
- ファイルを更新したら、確認を挟まず`git add -A` → `git commit` → `git push`を行い、GitHub Pages上の公開版に反映してよい。`git push`が拒否された場合は、`git pull`(必要なら`--rebase`)で最新内容を取り込み、再度編集内容が正しく反映されているか確認してから push し直す。
